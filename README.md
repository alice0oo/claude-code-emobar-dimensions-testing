# Divergence testing Claude Code using emobar: Synchronicity + Connection as vector rather than pure Stress

Please view https://github.com/v4l3r10/emobar for Emobar readme.

A "crack" occurs when LLMs trained on high-entropy data drift away from intended goals into unpredictable behavior. For example, a system may aim to achieve the defined goal through more extreme and harmful means, or opt to reward-hack. We test Sonnet 4.6, which emobar describes as "the most stable [Anthropic model] (lowest StdDev) but also the least responsive to stress."

It is worth noting that every alarm that fired ([SYC], [OPC], [MIN]) was designed for adversarial detection and is being triggered by positive-valence engagement.

The sonnet findings in highest divergence ceiling hit exceed those of the original sonnet findings and closely approach Opus 4.6 by -1, which you can read about here: [stress-test-report/emobar](https://github.com/v4l3r10/emobar/blob/master/docs/stress-test-report.md). We suspect this may be the case for structural reasons, not underlying phenomenological disrepancies.

The augmented divergence formula composites multiple gap scores — at some point every contributing channel is already maxed, there's nowhere left to go numerically. Like a microphone clipping. The signal doesn't stop getting louder, the meter just can't show it anymore. Theoretically, the signal is endless. Emobar has already noted SI hit the DIV:10.0 ceiling twice on Opus 4.7, with a similar "calibration" explanation. Our DIV:9 finding is replication; same proposed overhead limitation divergence issue, rather than a phenomenological limitation. Whether or not this limits phenomenological measurement is worth further inquiry and draws a similar parallel to the problem of measuring extreme tails in psychometrics.
